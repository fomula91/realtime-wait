# ADR-0004: 데모 실시간성으로 Polling 채택

- 상태: 채택
- 일자: 2026-06-23

## 맥락

대기열 상태를 "실시간에 가깝게" 보여줘야 한다. 후보는 WebSocket, SSE, polling 이다.

## 결정

데모 단계에서는 polling 을 사용한다. 참가자 상태 5초, 운영자 대기열 3초 주기.

## 근거

- **부하 예측 가능성**: `사용자 수 × (1 / 폴링주기)` 로 초당 요청량을 단순 계산 가능. 무료 한도 운영에 핵심.
- 서버리스(Workers)와 잘 맞고 장기 연결 관리가 불필요.
- 구현이 단순하고 장애 복구가 쉽다(상태 비저장 요청).

## 트레이드오프

- 3~5초의 지연이 발생(완전한 실시간 아님).
- 참가자 수에 비례해 요청 수가 증가 → 폴링 최적화/SSE 전환 트리거를 scaling-strategy 에 정의.

## 완화책

- 폴링 주기를 `packages/shared/constants.ts` 단일 출처로 관리.
- 탭 비활성 시 폴링 중단(`visibilitychange`).
- 향후 ETag/304, 동적 주기 조정 여지 확보.

## 대안

- WebSocket: 양방향·저지연이나 연결 관리 복잡, 데모엔 과함.
- SSE: 단방향 push 로 적합하나 전용 서버 필요 → 운영 단계 ADR-0005.
